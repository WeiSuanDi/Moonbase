# 🌙 月球基地（Moon Base）项目架构分析

> 分析日期：2026-06-19 | 版本：v1

---

## 一、项目概述

**月球基地畅想**（WEISUANDI Moon Base）是一个以月球南极基地建设为主题的**交互式 Web 产品**，部署在 Vercel 上。

核心体验：用户进入网站看到一颗可交互的 3D 月球，表面漂浮着若干真实命名地点的点选光标。用户点击一个地点作为基地选址，然后通过一步步的决策（能源、水源、辐射防护等）搭建出一套属于自己的基地配置，每个选择都会带来连锁后果。流程走完后可生成一份可行性简报。

**定位**：一件 "作品 / 游戏"，AI（agent）是其中关键位置的**增强**，不是主体。

## 二、技术栈总览

| 层级 | 技术 | 运行位置 |
|------|------|----------|
| **3D 渲染** | Three.js (r128) + UnrealBloomPass 后处理 | 用户浏览器 |
| **前端逻辑** | 原生 ES Module JS（无框架） | 用户浏览器 |
| **样式** | 纯 CSS（CSS Variables + Glassmorphism 风格） | 用户浏览器 |
| **后端** | FastAPI (Python) + Uvicorn | Vercel Serverless Function |
| **AI 模型** | DeepSeek (deepseek-chat) 通过 OpenAI SDK | Vercel / DeepSeek API |
| **状态持久化** | localStorage（浏览器端） | 用户浏览器 |
| **部署平台** | Vercel（静态托管 + Serverless） | Vercel 云端 |

## 四、架构设计原则

### 4.1 核心原则：确定性优先

判断一段逻辑该不该用 LLM 的标准：

> **"如果把 LLM 删掉，这部分会不会坏掉？"**
> - 会坏 → 用 agent（开放式推理、生成式产出）
> - 不会坏 → 用确定性代码（菜单、数值计算、状态更新、渲染）

### 4.2 物理分工

| 层 | 职责 | 删掉 LLM 后的表现 |
|----|------|-------------------|
| **前端（游戏本体）** | 3D 月球、点选、选项菜单、数值后果计算、状态管理、UI 渲染 | ✅ 照常运行 |
| **后端（agent 能力）** | ① 用户开放式追问 ② 收尾可行性简报生成 | ❌ 这两处会降级为本地占位 |

### 4.3 数据流

```
用户交互 → 前端 state.js（单一状态源）→ 规则表计算 → UI 重渲染
                  ↓ (需要 AI 时)
            agent-client.js → POST /api/agent 或 /api/summary
                  ↓
            FastAPI → DeepSeek API → 返回结果 → UI 展示
```

关键约束：**后端无状态** — Serverless Function 不保存任何会话记忆，前端每次请求必须携带完整 state。

## 五、核心模块详解

### 5.1 `scripts/state.js` — 单一状态源（SSOT）

**是整个项目除 agent 外最关键的文件。**

- **状态结构**：
  ```js
  baseState = {
    site: null,        // 选址 "shackleton"
    energy: null,      // "nuclear" | "storage" | "solar"
    water: null,       // "isru" | "earth_supply"
    radiation: null,   // "regolith" | "cave" | "hull"
    history: []        // 用户选择轨迹
  }
  ```

- **规则表（ruleTable）**：定义每个选项对 7 个指标的增量影响（功率平衡、质量、供水、功耗、风险、成本、可持续性），配合选址基础值计算最终推演结果。

- **computeMetrics()**：纯函数，根据 state + ruleTable + siteMeta 计算 6 个关键指标：
  - 能源结余 (kW)
  - 总部署质量 (t)
  - 年供水量 (t)
  - 年辐射剂量 (mSv)
  - 综合风险 (0-10)
  - 可持续性 (0-15)

- **持久化**：通过 `localStorage` 保存状态，`subscribe()` 模式通知 UI 更新。

### 5.2 `scripts/moon-render.js` — 3D 月球渲染

- 使用 **Three.js** 构建 3D 场景：
  - `SphereGeometry` 球体（桌面 512 段、移动端 256 段）
  - `MeshStandardMaterial` 结合颜色贴图 + 位移贴图 + 凹凸贴图
  - 雾效 (`FogExp2`) 模拟深空氛围
  - 外层辉光壳（Additive Blending 半透明球体）
  - 3000 颗星空粒子背景

- **光照系统**：主方向光（模拟太阳）+ 蓝色补光 + 青色边缘光 + 环境光

- **后处理**：EffectComposer + UnrealBloomPass（辉光效果）

- **4 个基地标记点**（浮动闪烁的光标）：
  | 基地 | 位置 | 类型 | 可选址 |
  |------|------|------|--------|
  | 沙克尔顿基地 | 南极 (-89.5°, 0°) | 极地科研前哨 | ✅ |
  | 静海纪念站 | (0.7°, 23.5°) | 文化旅游港 | ❌ |
  | 雨海采矿区 | (32.8°, -15.6°) | 工业采矿区 | ❌ |
  | 第谷观测台 | (-43.3°, -11.2°) | 科研观测站 | ❌ |

- **交互**：Raycaster 射线检测点击/悬停标记点，OrbitControls 轨道控制

### 5.3 `scripts/game.js` — UI 控制器

- **面板管理**：
  - 基地信息面板（点击标记弹出）
  - 游戏决策面板（右侧沙盘面板）
  - 结果模态框（可行性简报）
  - AI 聊天浮窗（右下角 FAB 按钮）

- **决策流程**：3 步顺序决策（能源 → 水源 → 辐射防护），每步必须先完成前一步

- **简报生成**：
  - 优先调用 `/api/summary`（AI 生成）
  - 失败则降级为 `buildLocalSummary()`（前端规则表生成）
  - 简单的 Markdown → HTML 转换

- **AI 聊天**：通过 `/api/agent` 实现开放式追问

### 5.4 `scripts/agent-client.js` — API 调用封装

- 极简封装（仅 30 行），两个导出函数：
  - `askAgent(state, question)` → POST `/api/agent`
  - `generateSummary(state)` → POST `/api/summary`
- 错误处理：抛出包含 HTTP 状态码的异常

### 5.5 `api/main.py` — FastAPI 后端

- **框架**：FastAPI + CORS 中间件
- **AI 调用**：OpenAI SDK 指向 DeepSeek API（`deepseek-chat` 模型）
- **3 个路由**：
  | 方法 | 路径 | 功能 | AI 依赖 |
  |------|------|------|---------|
  | GET | `/api/health` | 健康检查 | ❌ |
  | POST | `/api/agent` | 开放式追问 | ✅（无 key 时返回演示文本） |
  | POST | `/api/summary` | 可行性简报 | ✅（无 key 时返回占位文本） |

- **Domain Knowledge**：预置 Shackleton 环形山的领域知识作为 system prompt（替代 RAG）
- **静态托管**：本地开发时挂载根目录静态文件
- **安全**：API Key 通过环境变量注入（`.env.local` 仅本地，生产用 Vercel 环境变量）

### 5.6 `styles/main.css` — 视觉设计

- **设计语言**：深空主题 + Glassmorphism（玻璃拟态）
- **CSS 变量体系**：颜色、圆角、状态色统一管理
- **核心技术**：
  - `backdrop-filter: blur()` 玻璃效果
  - `conic-gradient` 旋转光晕动画
  - CSS Grid 响应式布局
  - 自定义动画（浮动、脉冲、旋转）
- **移动端适配**：768px 断点，游戏面板/聊天/时间线布局自适应

## 七、部署架构

### Vercel 配置 (`vercel.json`)

```json
{
  "functions": { "api/main.py": { "maxDuration": 30 } },
  "rewrites": [{ "source": "/api/(.*)", "destination": "/api/main.py" }]
}
```

- 根目录：静态文件直接托管
- `/api/*`：路由到 FastAPI Serverless Function（最大 30 秒）
- 环境变量 `DEEPSEEK_API_KEY` 在 Vercel 后台配置

## 八、当前状态与局限性

### 已完成
- ✅ 3D 月球渲染（颜色 + 位移贴图 + 辉光后处理）
- ✅ 4 个基地标记点（1 个可选址 + 3 个展示用）

- ✅ 前端规则表推演（不依赖 AI）
- ✅ 后端 FastAPI 接通 DeepSeek
- ✅ AI 开放式追问 + 可行性简报生成

### v1 明确不做
- ❌ 数据库 / 持久化存储（状态仅存 localStorage）

## 九、关键设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 前端框架 | 无框架（原生 JS） | 项目规模小，避免构建工具链复杂度 |
| 3D 库 | Three.js CDN (importmap) | 无需打包，直接 ES Module 引入 |
| AI 模型 | DeepSeek (deepseek-chat) | 性价比高，中文能力强 |
| 后端框架 | FastAPI | 轻量、高性能、Vercel 原生支持 |
| 状态管理 | 自制 SSOT + 订阅模式 | 比引入 Redux 等更轻量 |
| 部署 | Vercel | 静态 + Serverless 一站式，免费额度够用 |
| CSS 方案 | 纯 CSS（无预处理器） | 规模可控，CSS Variables 足够 |
| 后端状态 | 完全无状态 | Serverless 冷启动约束，由前端携带状态 |
